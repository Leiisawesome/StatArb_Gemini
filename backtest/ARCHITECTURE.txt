╔═══════════════════════════════════════════════════════════════════════════╗
║                         DESIGN CONTRACT                                   ║
╚═══════════════════════════════════════════════════════════════════════════╝

✅ MUST DO:
  • Use InstitutionalBacktestEngine as black box
  • Load config from YAML (no hardcoded params)
  • Inherit from BaseExperiment
  • Return ExperimentResult
  • Save structured output (JSON + CSV)

❌ MUST NOT DO:
  • Re-implement signal generation
  • Re-implement execution logic
  • Re-implement risk management
  • Calculate indicators directly
  • Update positions outside engine

╔═══════════════════════════════════════════════════════════════════════════╗
║                          BENEFITS                                         ║
╚═══════════════════════════════════════════════════════════════════════════╝

1. Clean Separation
   - Engine logic isolated in institutional_backtest_engine.py
   - Experiments are pure orchestration (< 150 lines each)
   - No duplication of trading logic

2. Maintainability
   - Easy to add new experiments (template + config)
   - Config changes don't require code changes
   - Clear responsibility boundaries

3. Reproducibility
   - All parameters in version-controlled YAML
   - Structured output for analysis
   - Audit trail for every run

4. Consistency
   - All experiments follow same interface
   - Standardized output format
   - Predictable behavior
