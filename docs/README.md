## Category Descriptions

### 01_quick_start/
Quick start guides, tutorials, and getting started documentation for new users.

### 02_architecture/
System architecture documentation, design decisions, and infrastructure analysis.

### 03_compliance_audits/
Rule compliance documentation, audit reports, and rule updates.

### 04_implementation/
Detailed implementation guides, code structure, and implementation patterns.

### 05_project_summaries/
Project milestones, summaries, and high-level progress tracking.

### 06_archived_phase_summaries/
Historical documentation from previous development phases.

### 07_testing/
All testing-related documentation:
- **integration/**: Integration test plans and verification reports
- **coverage/**: Test coverage analysis and test type explanations

### 08_analysis/
Analysis and investigation reports organized by topic:
- **signals/**: Signal generation logic, rate analysis, and fixes
- **dataframes/**: DataFrame structure, audits, and hardening
- **strategies/**: Strategy compliance reviews, test coverage, and clarifications
- **compliance/**: Compliance-specific analysis (reserved for future use)

### 09_reports/
General reports and summaries that don't fit into other categories.

## Finding Documentation

- **Quick Start**: See `01_quick_start/`
- **Architecture**: See `02_architecture/`
- **Rules & Compliance**: See `03_compliance_audits/`
- **Testing**: See `07_testing/`
- **Analysis Reports**: See `08_analysis/`
- **Implementation Details**: See `04_implementation/`

## Notes

- All documentation uses Markdown (`.md`) format
- Files are organized by category and topic
- Analysis reports are further categorized by subject matter
- Historical documentation is preserved in `06_archived_phase_summaries/`
